# 📝 국민대학교 KOSS 웹 프로젝트 스터디 기획서 - 초보자용

- **작성자**: 20233056 신예림
- **프로젝트명**: OurTime (아워타임) - 우리들의 빈 시간 찾기

## 1. 프로젝트 목적 (현실화)

- **웹 기초 역량 강화**: HTML/CSS 레이아웃과 JavaScript 데이터 핸들링 숙달
- **직관적 일정 관리**: 텍스트 위주의 할 일을 시간표(Grid) 형태로 시각화
- **쉬운 공유**: 복잡한 친구 맺기 대신, **'그룹 코드'**를 공유하여 함께 시간을 맞추는 방식 채택

## 2. 주요 기능 (난이도 조정)

### 기능 1: 나의 시간표 작성 (Local Storage 활용)

- **수정 포인트**: 서버/DB 없이 브라우저의 LocalStorage에 일정을 저장하여 로그인 없이도 내 일정이 유지되게 함 (구현 난이도 하)
- 클릭으로 일정 이름과 시간대 선택 후 저장

### 기능 2: 시각적 시간표 뷰 (Grid 레이아웃)

- **수정 포인트**: 복잡한 외부 라이브러리 대신, CSS Grid를 공부하며 직접 24시간 칸을 만들어보기 (레이아웃 학습에 최고)

### 기능 3: 그룹 공유 및 시간 겹치기 (JSON 데이터 활용)

- **수정 포인트**: 친구 추가/수락 시스템 대신, '내 일정 데이터 내보내기/불러오기' 기능으로 대체
- 친구의 일정 데이터(JSON)를 내 화면에 업로드하면 내 시간표 위에 친구 일정이 반투명하게 겹쳐 보임

## 3. 페이지 구성 (간소화)

- **메인 페이지**: 내 시간표 (가장 큰 비중)
- **입력 모달**: 일정 이름, 시작/종료 시간 선택창
- **공유 페이지**: 내 데이터를 파일로 저장하거나, 친구 데이터를 업로드하는 영역
- **결과 페이지**: 나와 친구의 일정이 겹쳐진 '공동 시간표' 화면

## 4. 사용자 흐름 (Step-by-Step)

1. **일정 만들기**: 메인 화면 접속 → 일정 추가 → 브라우저에 자동 저장
2. **데이터 내보내기**: `[공유하기]` 클릭 → 내 일정 데이터가 텍스트(JSON)로 복사됨
3. **시간 맞추기**: 친구의 데이터를 복사해서 `[친구 일정 추가]` 칸에 붙여넣기 → 화면에 내 일정과 친구 일정이 다른 색으로 겹쳐 보임 → 빈 칸 확인!

## 5. 사용 기술 (초보자 추천 조합)

- **Language**: HTML5, CSS3, JavaScript (Vanilla JS)
- **Styling**: CSS Flexbox & Grid (직접 레이아웃을 짜보며 실력 향상)
- **Storage**: `Window.localStorage` (서버 없이 데이터 저장 가능)
- **Tool**: VS Code, Live Server (실시간 수정 확인)

## 6. 개발 계획 (3주 집중 코스)

### 1주차: HTML/CSS 뼈대 잡기

- **월~수**: 24시간 세로축과 요일별 가로축이 있는 시간표 레이아웃 만들기
- **목~일**: 일정 입력창(모달) 디자인 및 '추가' 버튼 클릭 시 화면에 사각형 그리기

### 2주차: JavaScript 로직 구현

- **월~수**: 입력한 데이터를 배열에 저장하고, 새로고침해도 안 사라지게 localStorage 연결하기
- **목~일**: 저장된 데이터를 바탕으로 시간표 칸에 색칠하기(Render 기능)

### 3주차: 데이터 공유 기능 (핵심)

- **월~수**: 내 일정을 JSON 텍스트로 바꾸기 & 친구 텍스트를 넣으면 화면에 그리기
- **목~일**: 색상을 다르게 하여(내 일정은 파란색, 친구는 빨간색) 겹치는 부분 시각화 후 배포(GitHub Pages)
